# Development Notes

Этот документ описывает, как устроены часы и где менять основные параметры.

## Цель проекта

Проект сделан под старый iPad mini с iOS 9.3.5. Главный сценарий - открыть страницу один раз через интернет, добавить ее на домашний экран и дальше использовать как большие настольные часы без постоянного подключения к сети.

Поэтому код специально простой:

- один `index.html`;
- без сборщика;
- без внешних шрифтов и библиотек;
- без Service Worker;
- без современного JavaScript, который может не работать в Safari на iOS 9.

## Почему Application Cache

Safari на iOS 9 не поддерживает Service Worker. Для офлайн-режима используется старый механизм Application Cache:

```html
<html manifest="clock.appcache">
```

Файл `clock.appcache` перечисляет ресурсы, которые Safari должен сохранить. Сейчас там кэшируется только `index.html`.

Важное правило: после любого изменения страницы нужно увеличить версию в комментарии:

```text
CACHE MANIFEST
# v22
```

Например, заменить `# v22` на `# v23`. Без этого iPad может не понять, что нужно скачать новую версию.

## Устройство `index.html`

Файл состоит из трех частей:

- `style` - вся геометрия, цвета, темы, портретный режим и режим секунд.
- `body` - разметка сегментных цифр и панели настроек.
- `script` - часы, настройки, жесты, сохранение параметров и офлайн-обновление.

## Семисегментные цифры

Каждая цифра состоит из семи `span`:

```html
<span class="seg seg-a"></span>
<span class="seg seg-b"></span>
<span class="seg seg-c"></span>
<span class="seg seg-d"></span>
<span class="seg seg-e"></span>
<span class="seg seg-f"></span>
<span class="seg seg-g"></span>
```

Названия сегментов:

- `a` - верхний;
- `b` - правый верхний;
- `c` - правый нижний;
- `d` - нижний;
- `e` - левый нижний;
- `f` - левый верхний;
- `g` - средний.

JavaScript хранит карту включенных сегментов в `digitMap`. Например, `3` включает `a`, `b`, `c`, `d`, `g`.

## Главные CSS-настройки

Размер цифр без секунд:

```css
.digit
#colon
```

Размер цифр с секундами:

```css
body.show-seconds .digit
body.show-seconds #colon
body.show-seconds .seconds .digit
```

Положение часов на экране:

```css
.wrap
#clock
```

Портретный режим:

```css
@media (orientation: portrait)
```

Цветовые темы:

```css
body.theme-amber
body.theme-red
body.theme-green
body.theme-ice
```

Вариант `flat`:

```css
body.style-flat ...
```

`Flat` специально оставляет только цифры на черном фоне. У него нет панели, верхних или нижних декоративных линий.

Толщина сегментов:

```css
body.weight-bold ...
```

Текущий обычный вид считается `thin`. Режим `bold` делает горизонтальные и вертикальные сегменты шире.

## Настройки на iPad

Кнопка `+` открывает панель настроек. Доступные параметры:

- `Style`: `Retro` или `Flat`.
- `Theme`: `Amber`, `Red`, `Green`, `Ice`.
- `Weight`: `Thin` или `Bold`.
- `Seconds`: показывать или скрывать секунды.
- `Colon Blink`: мигать или не мигать двоеточием.
- `Night Dim`: автоматическое затемнение с `22:00` до `07:00`.

Настройки сохраняются в `localStorage` под ключом:

```text
clockSettings
```

Если настройки повреждены или отсутствуют, код возвращается к значениям по умолчанию.

## Жесты

- короткий тап по часам переключает тему;
- долгое нажатие по часам включает или выключает секунды;
- тап по `+` открывает или закрывает настройки;
- тап вне панели закрывает настройки.

## Защита от выгорания

Функция `updateShift()` раз в несколько минут смещает весь блок часов на 1 пиксель. Это почти незаметно, но помогает не держать одни и те же пиксели постоянно включенными.

Смещения описаны в массиве:

```js
shiftPositions
```

## Как обновить опубликованную версию

1. Изменить `index.html`.
2. Увеличить версию в `clock.appcache`.
3. Сделать commit.
4. Отправить изменения в GitHub:

```bash
git add .
git commit -m "Describe change"
git push
```

5. На iPad открыть страницу онлайн и обновить ее 2-3 раза.

## Отладка на iPad

Если изменения не видны:

- проверьте, что версия в `clock.appcache` увеличена;
- откройте страницу при наличии интернета;
- подождите 10-20 секунд;
- обновите страницу 2-3 раза;
- если нужно, закройте Safari и откройте часы с домашнего экрана заново.

## Ограничения

- Нельзя надежно отключить автоблокировку экрана из веб-страницы.
- Точные визуальные отступы лучше подгонять по скриншотам с самого iPad.
- GitHub Pages нужен только для первой загрузки и обновлений; после кэширования часы могут работать без интернета.
