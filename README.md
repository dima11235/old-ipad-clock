# Retro iPad Clock

Офлайн-страница с крупными ретро-часами для iPad. Работает и на старом iPad mini с iOS 9.3.5, и на современных моделях, и в обычном браузере на компьютере.

Адрес GitHub Pages:

```text
https://dima11235.github.io/old-ipad-clock/
```

## Что умеет

Пять типов индикатора, каждый эмулирует свой класс ретро-дисплея:

- `LED` - семисегментные светодиоды: скошенные сегменты, почти невидимые погашенные сегменты, многослойное свечение;
- `Nixie` - газоразрядные лампы с цифрами внутри: стеклянная колба, анодная сетка, светящаяся проволочная цифра, тени непогашенных катодов позади нее и лёгкое мерцание;
- `VFD` - вакуумно-люминесцентный индикатор: почти белое ядро сегмента внутри цветного ореола, светящиеся погашенные сегменты, нити накала поперёк экрана;
- `LCD` - жидкокристаллический экран с подсветкой: тёмные цифры на светящейся панели, видимые «призраки» сегментов, плавное переключение;
- `Flat` - минималистичный современный вариант без панели и свечения.

Плюс:

- цвет (`Amber`, `Red`, `Green`, `Ice`) - действует на LED, VFD, подсветку LCD и Flat; у газоразрядных ламп свой оранжевый неон. `Ice` в режиме VFD даёт классический бирюзовый вид;
- формат `24 ч` или `12 ч` с индикатором `AM/PM`;
- толщина сегментов `Thin` или `Bold`;
- секунды, дата с днём недели, мигание двоеточия;
- ночное автозатемнение с 22:00 до 07:00;
- язык интерфейса и даты: русский и английский;
- защита от выгорания: часы медленно смещаются на пару пикселей;
- офлайн-режим на всех поколениях iPad;
- на новых iPad экран не гаснет (Wake Lock API).

## Универсальный размер

Часы не привязаны к конкретной модели. При каждом запуске, повороте и изменении настроек JavaScript подбирает максимальную высоту цифры, при которой всё содержимое ещё помещается в текущий экран, и записывает готовые пиксельные размеры в служебный `<style id="metrics">`. Поэтому одинаково хорошо выглядят iPad mini 1024x768, iPad Air 1180x820, iPad Pro 1366x1024, портретная ориентация и окно браузера на компьютере.

## Файлы проекта

- `index.html` - вся страница часов: HTML, CSS и JavaScript в одном файле.
- `sw.js` - service worker, офлайн-кэш для современных iPad и браузеров.
- `clock.appcache` - манифест Application Cache для старого Safari на iOS 9.
- `.nojekyll` - говорит GitHub Pages отдавать файлы как есть, без Jekyll.
- `docs/IPAD-SETUP.md` - как настроить старый iPad под настенные часы.
- `docs/DEVELOPMENT.md` - устройство кода и принципы эмуляции индикаторов.
- `docs/VERIFICATION.md` - как проверить, что часы отображаются правильно.
- `tools/check.js` - сама проверка: раскладки, яркость, скриншоты. На iPad не попадает.

## Проверка после правки

```bash
cd tools
npm install
node check.js
```

Скрипт открывает часы в headless Chrome и проверяет то, что не проверяется глазами: влезают ли часы во все пять поколений iPad во всех режимах, и светится ли дата с той же яркостью, что и цифры. Подробности - в [docs/VERIFICATION.md](docs/VERIFICATION.md).

## Офлайн на трёх поколениях iPad

- Safari на iOS 9 не знает service worker и использует `clock.appcache`;
- Safari начиная с iOS 14 не знает Application Cache и использует `sw.js`;
- на устройстве всегда работает ровно один из двух механизмов, они не конфликтуют.

После изменения `index.html` нужно поднять версию **в обоих местах**: комментарий `# v57` в `clock.appcache` и `CACHE_NAME` в `sw.js`. Иначе iPad может продолжать показывать старую версию.

## Как использовать на iPad

1. Откройте опубликованный адрес в Safari.
2. Дождитесь полной первой загрузки при наличии интернета.
3. Добавьте страницу на домашний экран через `Share -> Add to Home Screen`.
4. Дальше часы открываются без интернета.

Чтобы iPad работал как настоящие настенные часы - не гас, не блокировался, сам приглушался ночью и не выпадал в другие приложения - см. [docs/IPAD-SETUP.md](docs/IPAD-SETUP.md).

## Управление на экране

- обычное нажатие по часам переключает цвет;
- долгое нажатие около 1 секунды включает или выключает секунды;
- кнопка `+` в углу открывает панель настроек;
- нажатие вне панели закрывает её.

## Быстрый локальный запуск

```bash
python -m http.server 8080
```

Потом откройте на iPad адрес вида `http://IP_ВАШЕГО_КОМПЬЮТЕРА:8080`. Service worker требует `http(s)`, при открытии файла напрямую он просто не регистрируется, а часы работают.

Подробнее о внутреннем устройстве проекта: [docs/DEVELOPMENT.md](docs/DEVELOPMENT.md).
