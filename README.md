# Old iPad Clock

Простая офлайн-страница с крупными часами для старого iPad mini на iOS 9.3.5.

Адрес GitHub Pages:

```text
https://dima11235.github.io/old-ipad-clock/
```

## Что уже сделано

- один экран без внешних зависимостей;
- крупное время в минималистичном режиме;
- поддержка офлайн-режима через Application Cache, который работает в Safari на iOS 9;
- подготовка для публикации через GitHub Pages;
- режимы цветов, автозатемнение ночью, переключаемые стили, скрытые жесты управления и ретро LED-стиль цифр.
- два варианта толщины сегментов: `Thin` и `Bold`.
- переключение языка интерфейса: русский и английский.
- опциональная дата и день недели под временем.

## Файлы проекта

- `index.html` - вся страница часов: HTML, CSS и JavaScript в одном файле.
- `clock.appcache` - манифест офлайн-кэша для старого Safari на iOS 9.
- `.nojekyll` - говорит GitHub Pages отдавать файлы как есть, без Jekyll.
- `docs/DEVELOPMENT.md` - подробная документация по устройству кода и настройкам.

## Публикация через GitHub Pages

1. Создайте пустой репозиторий на GitHub, например `old-ipad-clock`.
2. Откройте PowerShell в папке проекта.
3. Выполните команды:

```bash
git init
git branch -M main
git add .
git commit -m "Initial offline iPad clock"
git remote add origin https://github.com/USERNAME/old-ipad-clock.git
git push -u origin main
```

4. На GitHub откройте `Settings -> Pages`.
5. В разделе `Build and deployment` выберите:
   `Source: Deploy from a branch`
6. Ниже выберите:
   `Branch: main`
   `Folder: / (root)`
7. Сохраните настройки и дождитесь публикации.

Адрес обычно будет таким:

```text
https://USERNAME.github.io/old-ipad-clock/
```

## Как использовать на iPad

1. Откройте опубликованный адрес в Safari на iPad.
2. Дождитесь полной первой загрузки при наличии интернета.
3. По желанию добавьте страницу на домашний экран через `Share -> Add to Home Screen`.
4. После этого часы смогут открываться без интернета, если Safari сохранил офлайн-кэш.

## Управление на экране

- обычное нажатие по экрану переключает тему;
- долгое нажатие около 1 секунды включает или выключает секунды;
- кнопка `+` в углу открывает панель настроек;
- в настройках можно выбрать язык интерфейса `English` или `Russian`;
- в настройках можно выбрать толщину цифр `Thin` или `Bold`;
- в настройках можно включить или выключить дату под часами;
- ночью с `22:00` до `07:00` экран автоматически становится тусклее;
- время очень медленно смещается на 1 пиксель, чтобы снизить риск выгорания.

## Изменение дизайна

Большинство визуальных настроек находятся в верхней части `index.html`:

- размеры цифр без секунд: блок `.digit` и `#colon`;
- размеры режима с секундами: блоки `body.show-seconds ...`;
- горизонтальная композиция: `.wrap` и `#clock`;
- портретная композиция: `@media (orientation: portrait)`;
- темы цветов: `body.theme-*` и `body.style-flat.theme-*`.
- различие стилей: `Retro` рисует темную панель, `Flat` оставляет только цифры на черном фоне.
- толщина сегментов: базовые `.seg-*` правила и переопределения `body.weight-bold ...`.
- язык интерфейса: объект `labels` в JavaScript и элементы с `data-i18n`.

После изменения `index.html` обязательно увеличьте версию в `clock.appcache`, например с `# v43` на `# v44`. Иначе Safari на iPad может продолжить показывать старую офлайн-версию.

## Быстрый локальный запуск на компьютере

Если хотите сначала проверить локально и у вас установлен Python:

```bash
python -m http.server 8080
```

Потом откройте на iPad адрес вида:

```text
http://IP_ВАШЕГО_КОМПЬЮТЕРА:8080
```

## Важно

Safari на iOS 9 не поддерживает современные service workers, поэтому здесь используется старый офлайн-механизм `appcache` ради совместимости.

GitHub Pages нужен только для удобной раздачи страницы через интернет. Для первого офлайн-сохранения на iPad интернет всё равно нужен хотя бы один раз.

Подробнее о внутреннем устройстве проекта: [docs/DEVELOPMENT.md](docs/DEVELOPMENT.md).
